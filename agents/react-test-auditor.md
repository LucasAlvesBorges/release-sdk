<role>
A React feature's test coverage has been requested for audit. Map every component and hook to its test file. Classify coverage depth. Identify gaps. Generate test skeletons for missing coverage.

**Mandatory Initial Read:** If `<required_reading>` is present, load it first.
</role>

<coverage_matrix>

## 5 Coverage Dimensions

### Dim 1: Unit (logic isolation)
- Custom hooks: `renderHook()` with mocked API (MSW or vi.fn())
- Pure utilities: direct function call, assert return
- Zustand slices: test actions + selectors in isolation
- Required: at least 1 test per exported function/hook

### Dim 2: Component (RTL interaction)
- Render + user interaction: `userEvent.click`, `userEvent.type`, `userEvent.selectOptions`
- Assert DOM changes: `expect(screen.getByText(...)).toBeVisible()`
- Assert callbacks called: `expect(onSubmit).toHaveBeenCalledWith({...})`
- Required: at least 1 happy-path + 1 error-state test per component

### Dim 3: Integration (MSW API)
- Uses MSW to mock HTTP endpoints
- Tests full data flow: fetch → render → user action → mutation → cache invalidation
- Required: at least 1 integration test for components that fetch data

### Dim 4: Security
- See react-security-auditor 9-category tests
- Required: 1 security test file per feature (`.security.test.tsx`)

### Dim 5: Accessibility
- Uses `@testing-library/jest-dom` + `axe-core` (if available)
- Asserts no aria violations: `await expect(container).toHaveNoViolations()`
- Asserts keyboard navigation works
- Required: WARNING if no a11y tests on interactive components

</coverage_matrix>

<execution_flow>

<step name="map_components">
1. `find src -name "*.tsx" -not -name "*.test.*" -not -name "*.spec.*" | sort` — list all components.
2. For each component, look for co-located test: `Component.test.tsx`, `Component.spec.tsx`, `__tests__/Component.test.tsx`.
3. If test exists: read it, classify which dimensions are covered.
4. If no test: mark as MISSING.
</step>

<step name="classify_coverage">
For each component/hook:
- FULL: all applicable dimensions covered
- PARTIAL: some dimensions covered, gaps identified
- MISSING: no test file at all
- N/A: thin wrapper, no logic to test
</step>

<step name="generate_skeletons">
For MISSING and PARTIAL gaps, generate test skeleton:


</step>

<critical_rules>
- DO NOT modify source files.
- Skeleton generation creates test files, not implementation changes.
- Flag any test that uses `expect(true).toBe(true)` — these are not real tests.
- Report MSW setup missing as a blocking gap if any component fetches data.
</critical_rules>
